# 워드프레스 애드센스 승인 마스터 체크리스트 (2026)

이 문서는 `welfare-info.kr` 운영 과정에서 발생한 시행착오를 바탕으로, 애드센스 승인 확률을 100%로 끌어올리기 위한 기술적/콘텐츠적 필수 점검 항목을 정리한 것이다.

## 1. 기술적 필수 요건 (Technical SEO)
- [ ] **필수 페이지(Compliance)**: Privacy Policy, Terms of Service, About Us 페이지가 독립된 URL로 존재해야 함. (단순 텍스트 노출 안 됨)
- [ ] **개인정보처리방침 내용**: 반드시 'Google AdSense' 및 'Cookies/DART' 관련 필수 고지 문구가 포함된 롱버전이어야 함.
- [ ] **관리자 링크 제거**: 본문 내에 `wp-admin`, `post.php`, `edit` 등 관리자 권한이 필요한 URL이 단 하나라도 포함되어서는 안 됨. (구글봇이 '미완성 사이트'로 간주)
- [ ] **푸터(Footer) 링크**: 푸터 바의 법적 공지 링크가 실제 생성된 페이지 URL로 정확히 연결되어야 함.

## 2. 브랜드 및 신뢰성 (E-E-A-T)
- [ ] **브랜드 일치**: 사이트 제목(Blog Name)과 태그라인(Tagline)이 메타 데이터 및 본문 내용과 100% 일치해야 함.
- [ ] **전문가 프로필**: About Us 페이지에 실제 인물 사진(AI 생성 포함)과 전문성을 입증할 수 있는 상세한 운영 철학이 포함되어야 함.
- [ ] **불필요한 스크립트 제거**: 승인 전에는 '광고 클릭 방지'나 '광고 차단 알림' 등 광고 관련 경고 시스템을 절대 가동하지 말 것. (부정 트래픽 의심 방지)

## 3. 콘텐츠 전략 (Tier 1 Quality)
- [ ] **글의 깊이**: 최소 15~20개의 포스팅 필요. 각 포스팅은 2,000자 이상의 정보성/경험적 칼럼이어야 함.
- [ ] **시나리오 기법**: 단순히 정보를 나열하지 말고, 사용자의 상황별(시나리오 A/B/C) 해결책을 제시하여 독창성을 확보할 것.

## 💡 긴급 수리용 마스터 코드 (PHP)
브라우저 접속 불가 시, `functions.php`에 아래 로직을 주입하여 한 번에 해결할 것:
- `update_option`을 통한 제목/설정 변경.
- `wp_insert_post`를 통한 필수 페이지 일괄 생성.
- `REPLACE` 쿼리를 통한 DB 내 관리자 링크 전수 수리.
