# AI 1인 기업: AI 비서 구축 및 API 연동 가이드

이 문서는 AI 기반 1인 기업 운영을 위한 API 연동 로직과 데이터 기반 콘텐츠 자동화 전략을 담고 있다.

## 1. API 파이프라인 자동화의 가치
- **수동 수집의 한계**: 하루 3시간 소요, 휴먼 에러 발생, 실시간 업데이트 불가.
- **API 자동화의 이점**: 24시간 자동화, 정확한 데이터, 즉각적인 동기화.

## 2. API 인증 방식 결정 트리
- **OAuth 2.0**: 개인 정보(로그인) 접근 필요 시 사용. (Client ID/Secret)
- **서비스 계정 (Service Account)**: 서버 간 백그라운드 자동화 스크립트 최적. (JSON 키)
- **API Key**: 단순 공개 데이터 조회용. 보안에 유의.

## 3. 유튜브 API 생태계 및 할당량(Quota) 관리
유튜브 API는 하루 **10,000포인트**의 기본 제공량을 가진다. 효율적인 관리가 필수적이다.

### 할당량 소모 기준
- **가벼운 작업 (Read)**: 단순 리소스 조회 = **1점**
- **무거운 작업 (Search/Upload/Write)**: 검색, 영상 업로드, 수정 등 = **100점**
- **중요**: Search API를 무분별하게 호출하면 할당량이 즉시 고갈되므로, `playlistItems` 등 우회 루트를 활용한다.

### 핵심 리소스 매핑 (UI Anatomy)
- `channel`: 채널 정보 및 메타데이터.
- `video`: 단일 영상 통계 및 세부 정보.
- `search result`: 검색 조건 일치 항목 리스트.
- `commentThread`: 댓글 및 대댓글 데이터.

## 4. 구글 캘린더 API: 수석 비서실장
- **CalendarList / Calendar**: 캘린더 메타데이터 관리.
- **Event**: 제목, 시간, 참석자 등 핵심 일정 데이터 제어.

## 5. 실전 아키텍처: Python API 요청 구조
1. **인증 로직**: `client_secret.json` 로드 및 권한 동의 처리.
2. **서비스 객체 빌드**: `build('youtube', 'v3', credentials)`로 전용 Client 생성.
3. **요청(Request) 구성**: 필터(`part`, `fields`)를 적용하여 데이터 효율 극대화.
4. **실행 및 응답 처리**: `execute()` 호출 후 JSON 파싱 및 DB 저장.

## 6. 에이전틱 AI: 독립적 1인 기업의 핵심
- **Generative AI**: 작업 속도를 개선.
- **Agentic AI**: 당신(CEO)의 비전과 결정에 따라 독립적으로 작업을 오케스트레이션하고 추론함.
- **구조**: 당신(비전) -> 에이전틱 AI(추론/실행) -> 유튜브/캘린더 API(손과 발).

## 7. 실전 시나리오: 자율형 콘텐츠 전략가
1. **데이터 수집**: 유튜브 API로 틈새 시장 인기 영상 Top 50 수집.
2. **분석**: 제목 키워드, 댓글 감정 분석으로 성공 확률 계산.
3. **결과**: 일일 전략 리포트 및 콘텐츠 초안 완성.
