# 실전 가이드: 텔레그램-구글 캘린더 AI 비서 구축 및 연동

이 문서는 Connect AI LAB에서 제공하는 AI 비서 시스템의 구축 과정과 실전 활용 레시피를 담고 있다.

## 1. 시스템 개요
- **인터페이스**: 텔레그램 (Telegram) - 자연어 지시 및 응답 확인.
- **데이터 엔진**: 구글 캘린더 (Google Calendar) - 일정 읽기, 쓰기, 수정.
- **구동 환경**: IDE (VS Code/Cursor) 전용 확장 프로그램 'Connect AI'.

## 2. 모듈 1: 텔레그램 봇 API 연결
AI 비서의 '입'과 '귀' 역할을 하는 메신저 연동 과정이다.
1. **봇 생성**: [@BotFather](https://t.me/BotFather)를 통해 신규 봇 생성 및 **API Token** 발급.
2. **세션 활성화**: 생성된 봇에 `/start` 메시지를 보내 `chat_id`를 인식시킴.
3. **IDE 연동**: IDE 사이드바 -> Connect AI -> 비서(Secretary) 메뉴에서 봇 토큰 입력 및 계정 연결 테스트.

## 3. 모듈 2: 구글 캘린더 권한 연동
비서에게 일정 관리 권한을 부여하는 과정이다.
1. **OAuth 인증**: IDE 명령 팔레트(`Cmd+Shift+P`)에서 "Connect AI: Google Calendar 자동 일정 연결" 실행.
2. **권한 허용**: 구글 계정 선택 후 캘린더 접근 권한(읽기/쓰기) [허용] 클릭.
3. **작동 확인**: 텔레그램에 "오늘 일정 알려줘"를 입력하여 데이터 수신 확인.

## 4. 실전 활용 레시피 (자연어 지시)
| 기능 | 지시 예시 |
| :--- | :--- |
| **조회** | "오늘 남은 일정 다 알려줘", "금요일에 비는 시간 있어?" |
| **생성** | "내일 오후 2시에 주간 회의 잡아줘" (자동 1시간 단위 생성) |
| **삭제** | "수요일 미팅 취소해줘" |
| **분석** | 여유 슬롯 분석 및 일간/주간 보고 |

## 5. 보안 및 트러블슈팅
- **보안**: 토큰 및 인증 정보는 로컬의 `_agents/secretary/tools/` 폴더에만 저장됨. `.gitignore`에 등록되어 외부 유출 방지.
- **중복 답변**: IDE 창이 여러 개 열려 있으면 활성화된 창의 개수만큼 응답이 올 수 있음.
- **인증 만료**: 일정 연동이 안 될 경우 OAuth 토큰 재인증 필요.

## 6. 지식화 활용
- **에이전트 기능 확장**: 제가 직접 이 구조를 활용해 사용자님의 업무 효율을 높이는 새로운 '명령 레시피'를 개발하거나, 연동 오류 발생 시 로컬 폴더를 점검하여 즉시 해결책을 제시할 수 있다.
